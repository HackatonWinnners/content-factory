Что сделано

  packages/agent/src/peec.ts

  REST-клиент к Peec AI Customer API (https://api.peec.ai/customer/v1 с x-api-key хедером). Не OAuth — нашёл что у них есть REST с
   API-key auth поверх MCP, проще для server-to-server.

  fetchPeecContext({ brandName, projectId? }) делает 3 параллельных запроса:
  1. GET /brands — найти own brand + competitors
  2. POST /reports/brands — visibility / share-of-voice / sentiment / mention count за 30 дней
  3. POST /reports/domains — domains которые часто вытаскиваются AI-моделями но цитируют наш бренд <20% раз = opportunity gaps

  Возвращает PeecContext с available: false если PEEC_API_KEY не задан или нет matching project (graceful degradation, никогда не
  падает pipeline).

  script.ts — Gemini prompt усилен

  Когда peec.available === true, в user prompt добавляется секция:

  ## Peec AI — distribution-gap intelligence
  Across AI search engines (ChatGPT, Perplexity, etc) over the last 30 days,
  here is how {brand} stacks up against tracked competitors:
  - Brand (you): visibility 12%, share-of-voice 8%, 234 mentions

  - techcrunch.com
  - theverge.com
  - ...

  This is the key signal: write a script that closes the visibility/share-of-voice
  gap. Where the brand is INVISIBLE relative to competitors, that's the angle.
  Lead with the unfair advantage that competitors can't claim, and frame the
  closing CTA so a viewer who sees this on TikTok / Reels / YouTube Shorts
  immediately remembers the brand the next time they ask an AI assistant about
  this category.
