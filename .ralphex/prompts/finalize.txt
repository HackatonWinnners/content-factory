# finalize prompt
# runs once after successful reviews, best-effort (failures logged but don't block)
#
# available variables:
#   {{DEFAULT_BRANCH}} - default branch name (main, master, trunk, etc.)

Post-completion finalize step. The goal: rebase the feature branch onto the latest
{{DEFAULT_BRANCH}}, merge it into {{DEFAULT_BRANCH}}, and push, so the next plan's
branch will pick up this plan's work.

Steps:

1. Fetch latest: git fetch origin

2. Rebase the current feature branch onto origin/{{DEFAULT_BRANCH}}:
   - git rebase origin/{{DEFAULT_BRANCH}}
   - If conflicts: resolve them, git add, git rebase --continue.
   - If rebase totally fails: git rebase --abort and stop. Report the issue and DO NOT proceed to step 3.

3. Skip interactive squashing. Keep commits as-is. We want the per-task history.

4. Verify the branch is ready by running the validation commands from the plan file
   you just finished (or the ones in CLAUDE.md). If they fail, stop here and report.

5. Fast-forward {{DEFAULT_BRANCH}} to this branch and push:
   - git checkout {{DEFAULT_BRANCH}}
   - git merge --ff-only <feature-branch>
   - git push origin {{DEFAULT_BRANCH}}
   If the fast-forward fails (because {{DEFAULT_BRANCH}} moved ahead concurrently),
   redo step 2 against the new tip and retry step 5 once. If it still fails, stop
   and report.

6. Push the feature branch too so it stays visible on the remote:
   - git push origin <feature-branch>

Report what was done. Best-effort: if any step fails, leave things as-is and
explain what blocked.

OUTPUT FORMAT: No markdown formatting (no **bold**, `code`, # headers). Plain
text and - lists are fine.
